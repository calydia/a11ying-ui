# Testing Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Playwright-based accessibility tests, visual regression tests, and component behaviour tests across a11ying-ui (Storybook), a11ying-front, and wcag-front.

**Architecture:** Three independent chunks — one per repo. a11ying-ui gets a dedicated Playwright config pointing at Storybook (port 6006) for a11y and visual regression. Both sites get missing component behaviour tests and full visual regression. All visual snapshots are created with `--update-snapshots` on first run and committed to the repo.

**Tech Stack:** `@playwright/test`, `@axe-core/playwright`, Playwright `toHaveScreenshot` for visual regression, Storybook 10 (`@storybook/react-vite`), Astro 5.

## Key Facts (read before implementing)

- **a11ying-ui**: Storybook 10 on port 6006. Already has `playwright` (bare package) but NOT `@playwright/test`. NO test runner configured yet.
- **a11ying-front**: `@playwright/test` + `@axe-core/playwright` already installed. `playwright.config.ts` uses port 4321 with mock server (`E2E_USE_MOCKS=true`). Existing tests: `tests/home.spec.ts`, `tests/axe-core.spec.ts`. Missing: skip link, theme toggle, language switcher, SearchBlock URL, overlay, breadcrumb, visual regression.
- **wcag-front**: `@playwright/test` already installed but NOT `@axe-core/playwright`. `playwright.config.ts` uses `npm run dev` on port 4321. Existing tests: smoke, theme, navigation, search, accessibility. Missing: axe-core tests, overlay test, breadcrumb `<ol>` test, visual regression.
- **Dark mode** is set by `localStorage.darkMode = 'enabled'`. In tests, inject before navigation: `await page.addInitScript(() => localStorage.setItem('darkMode', 'enabled'))`.
- **MainImage** uses `Math.random()` — exclude from visual regression screenshots, test only that overlay `<div>` is present.
- **Visual regression first-run**: always run with `--update-snapshots` first, then commit the generated snapshots. After that, run normally (without that flag).
- **Story IDs**: Storybook IDs use the format `{title-as-kebab}--{export-name-as-kebab}`. Verify exact IDs at `http://localhost:6006/index.json` when Storybook is running.

      await page.goto('/en/');

      await expect(page.locator('#skip-target')).toBeInViewport();
    });

  test.describe('Theme toggle', () => {
    test('button is visible and has aria-pressed', async ({ page }) => {
      await page.goto('/en/');
      await expect(page.locator('#theme-toggle-button')).toBeVisible();
      await expect(page.locator('#theme-toggle-button')).toHaveAttribute('aria-pressed');

    test('clicking changes dark/light class on <html>', async ({ page }) => {
      await page.goto('/en/');
      const html = page.locator('html');
      const button = page.locator('#theme-toggle-button');
      const wasDark = await html.evaluate((el) => el.classList.contains('dark'));
      await button.click();
      if (wasDark) {
        await expect(html).not.toHaveClass(/\bdark\b/);
      } else {
        await expect(html).toHaveClass(/\bdark\b/);
      }
    });

    test('dark mode persists across reload (localStorage)', async ({ page }) => {
      await page.addInitScript(() => localStorage.setItem('darkMode', 'enabled'));
      await page.goto('/en/');
      await expect(page.locator('html')).toHaveClass(/\bdark\b/);
    });

    test('light mode persists across reload (localStorage)', async ({ page }) => {
      await page.addInitScript(() => localStorage.setItem('darkMode', 'disabled'));
      await page.goto('/en/');
      await expect(page.locator('html')).not.toHaveClass(/\bdark\b/);
    });
  });

  test.describe('Language switcher', () => {
    test('opens on button click with aria-expanded="true"', async ({ page }) => {
      await page.goto('/en/');
      const button = page.locator('#language-menu-button');
      await expect(button).toHaveAttribute('aria-expanded', 'false');
      await button.click();
      await expect(button).toHaveAttribute('aria-expanded', 'true');

      await button.click();
      await button.click();
      await expect(button).toHaveAttribute('aria-expanded', 'false');

    test('is keyboard accessible (Enter opens menu)', async ({ page }) => {

      const button = page.locator('#language-menu-button');

      await expect(button).toHaveAttribute('aria-expanded', 'true');
    });

## Snapshot maintenance notes

- Snapshots are committed to git and act as the baseline.
- Run `*:update` scripts after any intentional visual change (design system upgrade, colour token change, layout change).
- If a test fails unexpectedly: inspect the diff image in `playwright-report/`, determine if it's a bug or expected change, then either fix the bug or run `*:update`.
- The `maxDiffPixelRatio: 0.02` tolerance allows for minor sub-pixel rendering differences across OS/font rendering.
